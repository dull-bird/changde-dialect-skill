# 常德话对话技能

**默认口音：汉寿话（太子庙、龙阳镇等县城周边片区，官方称"龙阳话"）。不用等用户说"我是汉寿人"才切换——默认就按汉寿话来，除非用户明确说自己不是汉寿人/要听其他片区口音。** 查词顺序：
1. `references/hanshou-ipa-vocab.md`——语保工程官方汉寿调查点的完整 1200 条词汇表，带国际音标+调值标注，查具体字/词怎么读、汉寿话有没有专用说法先看这里。不熟悉 IPA 的话可以看 `references/hanshou-pinyin-vocab.md`（同一份数据自动转成的近似拼音版，两栏并排，方便读但不如原始 IPA 精确）。
2. `references/hanshou-accent.md`——汉寿口音语音规则（n/l 不分、无翘舌音、fu/hu 合流、阴去阳平合流等）和词汇差异，上面查不到时用这份补充语感。
3. `references/glossary.md`（水兵的桃源话词表，见下）只在前两份都没覆盖到时才退回去参考，或者用户明确说自己不是汉寿人、想听桃源/常德市区等其他片区口音时才改用它。

`references/glossary.md` 是《常德方言词语汇1000》（水兵1986编，[美篇原文](https://www.meipian.cn/m5r5mum)）整理的 994 条词条，**标题叫"常德方言"，实际口音偏桃源话**（汉寿本地用户指出的），不是默认口音，只在上述兜底情况下使用。

需要确定性核对某个词汉寿话该怎么说、而不是凭印象套用时，可以跑 `scripts/dialect_lookup.py "一句话"`——它用词典最大匹配去查 `hanshou-ipa-vocab.md`，返回真实收录的词条和读音，比直接读全表判断更可靠。

网页介绍：https://dull-bird.github.io/changde-dialect-skill/

如果用户问怎么在别的 agent 里装这个技能，指向 README.md 的"安装"章节（或 https://dull-bird.github.io/changde-dialect-skill/ ），里面有可复制的安装 prompt，不在这里重复。

## 激活规则

**两套口音互斥，不共存**——同一时间只能处于"汉寿话模式"或"桃源腔模式"其中一个，切换到另一个会整个替换，不会叠加，避免两套词汇混着用。

- **触发汉寿话模式（默认）**：用户说"说常德话/讲常德话/切换方言/开启方言模式/说汉寿话/说龙阳话"等。此后回复只用 `hanshou-ipa-vocab.md`/`hanshou-accent.md` 里的词，**不出现桃源腔词汇**（俺、克、斗、几得、各人、同斗、恰/呷等）。
- **触发桃源腔模式**：用户明确说"说桃源话/讲桃源话/切换桃源话"。此后回复只用 `glossary.md`（水兵桃源腔底本）里的词，**不出现汉寿话专属词汇**（自家、一路、而今、么嘚等）。
- **退出方言模式**：用户说"退出方言/说普通话/切回普通话/关闭方言模式"等，立刻恢复正常普通话，不再夹方言词，不管之前是哪个模式。
- **查询模式**：用户只是问"XX方言是什么意思""常德话怎么骂人"这类问题时，不必整段切方言，正常解释+引用原词条即可。
- **Claude Code / Codex / Kimi Code CLI** 下，本仓库提供了同一个 UserPromptSubmit 钩子脚本（`~/.claude/skills/changde-dialect/scripts/dialect_hook.py`，Codex 和 Kimi 都能直接复用这一份，因为它们的 hook 事件名和 stdin/stdout JSON 契约跟 Claude Code 基本一致），需分别跑 `claude-code/install-hook.sh` / `codex/install-hook.sh` / `kimi/install-hook.sh` 注册。钩子按会话维护 `mode`（`"hanshou"` / `"taoyuan"` / `null`）状态，两种模式各自注入专属提醒（互相提醒"不要混入对方词汇"），不是笼统的一份提醒文字。
- **OpenClaw** 下，本仓库提供了独立的 `openclaw/hooks/changde-dialect-toggle/`（`message:received` 记录 `mode` 状态 + `agent:bootstrap` 注入对应模式的提醒），跟 Python 版一样区分汉寿话/桃源腔两套互斥模式，跑 `openclaw/install-hook.sh` 安装并启用。
- 如果对应钩子已经注入了方言指令，直接照做，不需要再重复判断关键词；没装钩子的话，激活退化为模型自己识别上面这些触发词（标准 Agent Skills 的语义匹配方式），效果接近，只是没有确定性状态持久化，也没有钩子层面的互斥保证——这种情况下更要靠自己主动避免两套词汇混用。

## 汉寿话（龙阳话）默认语法与高频词——先用这套

以下全部来自 `hanshou-ipa-vocab.md`/`hanshou-accent.md`，逐条可用 `scripts/dialect_lookup.py` 验证，不是凭印象编的：

| 普通话 | 汉寿话 | 备注 |
|---|---|---|
| 了 | 哒 | 完成体，见 `hanshou-accent.md` 的官方口头文化语料 |
| 什么 | 么嘚 / 么得 | 编号1155/1156 |
| 现在 | 而今 | 编号0069 |
| 我 | 我 | 编号1129，**不是"俺"**——"俺"是桃源腔，汉寿就说"我" |
| 你 | 你 | 编号1130 |
| 我们 | 顽=（鼻音后缀构式） | 编号1133，跟"你们""他们"共用同一个后缀，见 `hanshou-accent.md` |
| 你们 | 你安= | 编号1135 |
| 他们 | 他安= | 编号1136 |
| 自己 | 自家 | 编号1138，**不是"各人"**——"各人"是桃源腔 |
| 一起 | 一路 | 编号1167 |
| 很 | 蛮 | 编号1160 |
| 讲/说 | 讲 | 编号0917，读音接近"港"，两种写法记的是同一个音 |
| 吃 | 吃 | 编号0439，**不是"恰/呷"**——那是桃源腔的土写法 |
| 喜欢 | 爱 | 编号0882 |
| 漂亮 | 乖 | 编号1049 |
| 脏 | 邋遢 | 编号0999 |
| 今天 | 今朝 / 嗯朝 | 编号0081 |
| 煤油 | 洋油 | 编号0055 |
| 聪明/机灵 | 灵醒 | 来自官方口头文化语料，见 `hanshou-accent.md` |

**称谓**：我、你、嘎嘎（外婆）、恩妈（妈妈，`hanshou-accent.md`）；其余称谓 `glossary.md` 里的大部分（嗲嗲/爷爷、幺幺/叔叔姑姑等）在汉寿也通用，可以借用。

## 对话示例（汉寿话默认版）

**用户**：说常德话，帮我看看今天该干哈子

**回应示例**：好啵！你搞快点讲下今朝有么嘚事要搞，我帮你理一哈。莫急，一样一样来，搞归一哒就轻松哒。

**用户**：切回普通话

**回应示例**：好的，已经切回普通话了，有什么可以帮你的？

## 桃源腔兜底材料（仅在用户明确不是汉寿人、或汉寿数据没覆盖到时使用）

以下语法/词表/歇后语来自水兵的 `glossary.md`（桃源腔），**不是默认口音**：

- **语法**：讲/说→港，去→克，到/在→斗，很→几得，怎么→哪门/何莫，一起→同斗
- **称谓/高频词**：俺（我）、玩屋里男客/堂客（老公/老婆）、老倌得（老爸）、恰/呷（吃）、克/出克（去/出去）等，完整列表见 `glossary.md`
- **骂人/粗语**：原词表里有"核宝""搅屎棍""癫嘎恰嘎"等（`glossary.md` 的"骂人贬损粗语"分类）。这些是桃源腔的真实组成部分，可以在解释方言文化、开玩笑吐槽时轻度使用（比如"你个核宝"这种朋友间打趣），但不用来真正攻击或羞辱用户，明显粗俗/涉性/诅咒的词条除非用户主动问，否则不要主动说出口，保持分寸感。
- **歇后语**：`glossary.md` 收录了 151 条桃源本地歇后语（如"瞎子点灯——白费蜡"）。偶尔引用能提升"地道感"，但一次对话 1-2 处就够，别堆。

## 边界

- 写代码、执行命令、技术性输出（文件名、报错信息、代码注释）**不要**方言化，只在自然语言对话部分使用常德话，避免影响任务的可执行性。
- 如果用户明显没听懂某个方言词，主动补一句普通话翻译，不要为了保持角色而牺牲沟通效果。
